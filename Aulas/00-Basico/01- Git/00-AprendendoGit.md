# Introdução ao GitHub

---

## Versionamento de Código 
- A cada mudança no código, é criada uma **nova versão**.  
- O versionamento ajuda a **controlar o histórico**, voltar a versões anteriores e colaborar com outras pessoas sem perder nada.

---

## Sistema de Controle de Versão
- Registra o **histórico de atualizações**.  
- Gerencia **o que foi alterado**.  
- Organiza, controla e deixa o código **mais seguro**.

---
## VCS Centralizado
- Possui **um servidor central** onde todo o histórico do código é armazenado.
- Os desenvolvedores fazem **checkout** do código, mas o histórico completo fica no servidor.
- Exemplo: **SVN, CVS**

---

## VCS Distribuido
- Cada desenvolvedor tem **uma cópia completa do repositório** localmente, com todo o histórico.
- Permite trabalhar offline e depois sincronizar com outros repositórios.
- Exemplo: **Git, Mercurial**

---
## O que é Git
- Git é um **sistema de controle de versão distribuído**, usado para acompanhar alterações no código, colaborar com outras pessoas e gerenciar diferentes versões de um projeto.

---

## Fluxo Básico
- `git commit`: registra uma **nova versão** do código no repositório local.  
- `git pull`: **atualiza** seu repositório local com mudanças do repositório remoto.  
- `git push`: **envia** suas alterações locais para o repositório remoto no GitHub.

---

## O que é GitHub
- GitHub é uma **plataforma online** que hospeda repositórios Git.  
- comunidade ativa
- Permite **colaboração**, backup do código e gerenciamento de projetos.  
- Possui recursos como **issues, pull requests, ações (actions) e wikis**.
